It's day #3 of my 100 days of study!

Today we're learning about <a> anchor elements, <img> image elements, their attributes and values, and navigating our folder structure.
That might sound a little dry, so I've decided to reward your patience (and mine) with some sorcerous kittens!


Anyone who ever owned a Myspace back in it's heyday may be very familiar with the following HTML:

<a href=" "> </a>

<img src=" ">


Let's take a closer look at each of these, starting with <a href=" "> </a>

Here, the <a> tag refers to the anchor element - one of the most fundamental and widely used elements in HTML. Its primary purpose is to create hyperlinks, which are connections from one resource to another.

The href=" " that follows is the hyperlink reference and it is an attribute of the anchor element that takes a value.

You can probably guess that the content inside of the " " is the attribute's value which, in this case, specifies the destination URL of the hyperlink reference created by the element.


So.. putting this all together:

  • The <a> element defines its content as an anchor.
    <a>anchor</a>
    • The browser recognizes this as an anchor, but that's about it

  • The href=" " attribute establishes that the content is a clickable hyperlink - that it can be interacted with to navigate to another resource.
    <a href="">hyperlink</a>
    • The browser recognizes this as an anchor with a clickable hyperlink, but there is no destination

  • And the value entered between the quotation marks " " is the destination that the hyperlink navigates to. Click the link to navigate to the second page of the Day 3 content!
    <a href="./pages/anchor.html">Day 3, page 2!</a>
    • The browser recognizes this as an anchor with a clickable hyperlink
