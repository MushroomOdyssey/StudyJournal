Believe it or not, the absolute bare minimum for a valid HTML5 webpage is just two lines: a DOCTYPE declaration and a <title> element:

<!DOCTYPE html>
<title>?</title>

Those angle brackets < > are markup, instructions to be interpreted by the browser instead of content to be read by us.

<!DOCTYPE html> is a declaration that tells the browser: “Hey! You're looking at an HTML5 document. Render accordingly.”
<title>Study Journal</title> is an element - a start tag, some content, and an end tag. This content shows up in the browser tab.


That's it: a tiny, valid HTML document that demonstrates the fundamental building blocks: markup, tags, and elements.
The page is valid, though empty, and browsers will happily fill in the missing pieces for you:

<html> </html> - the root element, wrapping everything else
<head> </head> - holds metadata like the title, character encoding, and styles
<body> </body> - the container for visible content

Most elements come in pairs, but some void elements exist alone and don't need a closing tag.
