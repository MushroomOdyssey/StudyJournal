What is the absolute bare minimum required to create a valid HTML webpage? It may surprise you, but all it takes is a small collection of 2 lines of HTML composed of 1 declaration, 1 HTML element, and at least 1 character.


It looks like this:

<!DOCTYPE html>
<title>?</title>

Granted, all that will get you is an empty page, but it is a perfectly valid empty page.

This can be confirmed by checking the code in the Nu HTML Checker created by the World Wide Web Consortium (W3C) - a non-profit organization responsible for developing the standards that are used across the whole of the Web.


Let's take a look at each piece of this proto-webpage's HTML:


< >
The angle brackets < > are fundamental to HTML syntax. Anything inside of a pair of angle brackets is considered markup.
It's in the name! [H]yper[T]ext (one word) [M]arkup [L]anguage.
Anything defined as markup is interpreted by the browser as instructions, and not content.
You can imagine that using < > is somewhat like speaking to a dog. For the most part, you speak in a normal conversational tone to and around a dog and it will continue to happily go about its day while you do. But when you want your dog to do something, you may change your body language, alter your voice slightly, and enunciate your words more clearly:

Sit.
The dog sits.

Walk?
The dog fetches its leash.

Shake!
The dog lifts its paw for a handshake.

If your dog is trained and well-behaved (which is unlikely), it will perform these commands on cue.
But if you don't adopt this change in communication style, your dog will likely not assume that it should act on these words.

"I'm sorry Bob, I know it's what you want but I'm not going to roll over on this."
The dog does not begin rolling cross the living room.

It is the same with the browser. When we speak in a <tone> that it understands, it realizes "oh, you're talking to me!" and executes your commands.

We will see this play out below.


<!DOCTYPE html>
Now that we understand the function of the angle brackets, we can immediately see that this line is a set of instructions meant to be read and interpreted by the browser, not us.
Even though this line might appear at first glance to be composed of two parts, !DOCTYPE and html, it is actually a combination of three distinct instructions: !, DOCTYPE, and html.

! lets the browser know that what immediately follows is a declaration, which imparts metadata or additional rules to its target. The target of a declaration could be the entire page, or just a small selection of text.
Declarations are rare in HTML, though every valid webpage must include the one mentioned here.

DOCTYPE informs the browser that you are about to tell it what type of document it's looking at.
See below.

html, when used as part of the !DOCTYPE declaration, signals to the browser that it is looking at an HTML document which it should interpret using the HTML version 5 standards.
Other versions of HTML had more complicated values here, such as version 4.01, which required values that looked more like this:
HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd"


<title> </title>
Here we have another set of instructions, called an element. But you will notice some distinct differences between this element and the declaration above.

When the browser sees text enclosed in a < > pair, it will assume by default that the encapsulated text will be an element tag. That ! pre-pended to DOCTYPE exists speficically to yell at the browser: "I'M NOT AN ELEMENT TAG!.. I'm a declaration."
But what is a tag and what part of an element does it occupy? ...and what the hell even is an element??
I know, I know. I hate using terms before they've been defined. Unfortunately, there are so many of them and their uses and definitions are so woven into and amongst each other, I wouldn't even know how to begin without losing your attention entirely. So bear with me and we'll straighten this all out.

A tag is an instruction that defines the starting and ending boundaries of an element.
Many tags come in pairs, each of which is itself enclosed by a pair of delimiting symbols.
The < symbol is called the start tag open delimiter and it marks exactly where a start tag, the first half of a tag pair, begins.
The </ sequence of symbols is called the end tag open delimiter and it marks exactly where an end tag, the second half of the tag pair, begins.
And the > symbol is called the tag close delimiter and it marks exacty where any tag ends.
[start tag open] [tag] [tag close] content [end tag open] [tag] [tag close]
< start tag > content </ end tag >

An element is just the term used to describe a tag pair and its content: <title>Study Journal: Day 1</title>

As mentioned above, most tags come in a pair which enclose the content that the tag is modifying.
The simplest way to represent this is as follows:

...content... Begin instruction here--<tag> modified content </tag>--End instruction here ...content...

You've already witnessed the effect that a tag pair has on it's content many times on this page.

...content... modified content ...content...

But some tags exist alone, without the need for an end tag, and make up an entire element by themself.
Dramatically named void elements, they do not modify any content and therefor do not need an end tag. These can also be referred to as self-closing tags, if you have a preference for dull conversation.


?
At lease one character in length, this ? could have been anything, but the content of <title> </title> must not remain blank or empty.


Not too bad, right? But.. why did I call it a proto-webpage at the start of this journal entry?
Well, that's because the browser is actually doing some work here filling in a lot of blanks.
Browsers kind of know what to expect from HTML and their method for understanding an HTML document implies the existence of a few more elements that the browser infers if not stated in the document.

So when we write:

<!DOCTYPE html> -- tells the browser which HTML standards to use
<title>?</title> -- gives the page a name shown in the browser tab

The browser infers the following:

<!DOCTYPE html>
<html> -- the root element of the webpage; all other elements are children of <html>

<head> -- holds the webpage's metadata: title, character encoding, styles, etc
<title>?</title>
</head>

<body> -- contains the actual content that is visible on the page - what you are reading right now!
</body>

</html>


There is obviously still a lot to learn about this basic structure, the HTML boilerplate, but we now have a general understanding of basic HTML syntax and the fundamental building blocks of webpage creation.
