What is the absolute bare minimum required to create a valid HTML webpage? It may surprise you, but all it takes is a small collection of 6 lines of HTML composed of 1 declaration, 3 HTML elements, and 1 character.


It looks like this:

<!DOCTYPE html>
<html>

<head>
    <title>?</title>
</head>

</html>

Granted, all that will get you is an empty page, but it is a perfectly valid empty page.

This can be confirmed by checking the code in the Nu HTML Checker created by the World Wide Web Consortium (W3C) - a non-profit organization responsible for developing the standards that are used across the whole of the Web.


Let's take a look at each piece of this proto-webpage's HTML:


< >
The angle brackets < > are fundamental to HTML syntax. Anything inside of a pair of angle brackets is considered markup.
It's in the name! [H]yper[T]ext (one word) [M]arkup [L]anguage.
Anything defined as markup is interpreted by the browser as instructions, and not content.
We will see this play out below.


<!DOCTYPE html>
Now that we understand the function of the angle brackets, we can immediately see that this line is a set of instructions meant to be read and interpreted by the browser, not us.
Even though this line might appear at first glance to be composed of two parts, !DOCTYPE and html, it is actually a combination of three distinct instructions: !, DOCTYPE, and html.

! lets the browser know that what immediately follows is a declaration, which imparts metadata or additional rules to its target. The target of a declaration could be the entire page, or just a small selection of text.
Declarations are rare in HTML, though every valid webpage must include the one mentioned here.

DOCTYPE is an abbreviated form for document type. Its modern use is strictly to inform the browser that the instructions that follow it should be interpreted as some version of HTML.
See below.

html, when interpreted as part of the !DOCTYPE declaration, simply identifies that the HTML document should be interpreted using HTML version 5 standards. Previous versions of HTML had more complicated values here, but thankfully we shouldn't have to worry about that for now.
